# الزّهراء / AL-ZHRAA MODEST WEAR

مشروع storefront عربي RTL لبيع الملابس الإسلامية النسائية، مبني بـ HTML + CSS + JavaScript، ويعتمد على Supabase للمنتجات والـ variants والطلبات ولوحة الإدارة.

## الحالة الحالية الفعلية

هذه النسخة هي النسخة العاملة الحالية للمشروع بعد تحويله من منطق الصابون إلى منطق ملابس يعتمد على البيانات من Supabase:

- `index.html` + `script.js`: الصفحة الرئيسية وعرض المنتجات من قاعدة البيانات
- `product.html` + `product.js`: صفحة المنتج واختيار اللون والمقاس والصور والسعر حسب الاختيار
- `cart.html` + `cart.js`: السلة وإرسال الطلب
- `account.html` / `login.html` / `signup.html` / `my-orders.html`: مسارات الحسابات والطلبات
- `admin-login.html` + `admin-orders.html` + `admin-orders.js`: لوحة الإدارة الحالية للطلبات
- `catalog-data.js`: طبقة مساعدة مشتركة لتحميل المنتجات والصور والـ variants من Supabase
- `style.css`: التنسيقات المشتركة
- `supabase-clothing-schema.sql`: ملف SQL المطلوب لتجهيز الجداول الجديدة/المعدلة في Supabase

## ما الذي تغيّر فعليًا

### المتجر
- الصفحة الرئيسية لم تعد تعتمد على حقول الصابون القديمة.
- المنتجات الآن تعتمد على:
  - `products`
  - `product_variants`
  - `product_images`
- البطاقة تعرض:
  - اسم المنتج
  - الفئة
  - أقل سعر متاح من الـ variants
- الفلاتر تتكون من الفئات الموجودة فعلًا في قاعدة البيانات.

### صفحة المنتج
- تم إضافة صفحة منتج مستقلة: `product.html`
- كل منتج يمكن أن يحتوي على:
  - ألوان
  - مقاسات
  - صور لكل لون
  - سعر مختلف حسب الـ variant
- عند تغيير اللون أو المقاس تتغير:
  - الصور
  - السعر
  - حالة التوفر

### السلة والطلب
- السلة أصبحت تحفظ:
  - `product_id`
  - `variant_id`
  - الاسم
  - اللون
  - المقاس
  - السعر
  - الصورة
  - الكمية
- الطلبات المخزنة في `orders.items_json` أصبحت تحمل تفاصيل الاختيار الفعلي، وليس اسم المنتج فقط.

### طلباتي وإعادة الطلب
- إعادة الطلب أصبحت تحاول إعادة بناء السلة من:
  - المنتج الحالي
  - الـ variant الحالي
- وإذا لم يعد الـ variant متاحًا، يتم التحذير وإضافة المتاح فقط.

## الجداول المستخدمة في Supabase

### `products`
الجدول الرئيسي للمنتج نفسه.

الحقول التي تستخدمها الواجهة الآن:
- `id`
- `slug`
- `name`
- `category`
- `short_description`
- `description`
- `long_description`
- `fabric`
- `care_note`
- `size_guide`
- `sort_order`
- `is_active`

> يمكن أن تبقى الحقول القديمة الخاصة بالصابون موجودة في الجدول بدون مشكلة، لكن الواجهة الجديدة لا تعتمد عليها.

### `product_variants`
يمثل كل صف اختيارًا فعليًا من:
- لون
- مقاس
- سعر
- مخزون

الحقول المستخدمة فعليًا:
- `id`
- `product_id`
- `size_label`
- `color_name`
- `color_hex`
- `price`
- `compare_price`
- `stock_quantity`
- `sku`
- `sort_order`
- `is_active`

### `product_images`
صور المنتج المرتبطة باللون.

الحقول المستخدمة فعليًا:
- `id`
- `product_id`
- `color_name`
- `image_path`
- `alt_text`
- `sort_order`
- `is_active`

> `image_path` يمكن أن يكون:
> - رابطًا مباشرًا كاملًا `https://...`
> - أو مسار ملف داخل bucket `products` مثل `khimar/black-1.jpg`
> - أو `products/khimar/black-1.jpg`
>
> الواجهة تجلب الصورة من قاعدة البيانات، وليس من مسار ثابت داخل الكود.

### `orders`
ما زال هو الجدول الذي يستقبل الطلبات من `cart.js` ويُحدّث من لوحة الإدارة.

### `orders_dashboard`
ما زال يُستخدم للعرض داخل لوحة الإدارة.

## ملف SQL المطلوب

نفّذ الملف التالي في Supabase SQL Editor:

- `supabase-clothing-schema.sql`

هذا الملف يقوم بـ:
- إضافة الأعمدة الجديدة المطلوبة إلى `products`
- إنشاء `product_variants`
- إنشاء `product_images`
- تفعيل سياسات القراءة العامة للواجهة
- تجهيز bucket الصور `products` لو لم يكن موجودًا
- تعطيل المنتجات القديمة افتراضيًا حتى تبدأ بإضافة المنتجات الجديدة بنفسك من Supabase

## ملاحظات عملية مهمة

- لو لم تضف أي منتجات/variants/images في Supabase فالمتجر سيظهر فارغًا — وهذا مقصود.
- لا تحتاج تعديل الكود كل مرة تضيف فيها منتجًا جديدًا.
- يكفي أن تضيف البيانات الصحيحة في الجداول الجديدة ويظهر المنتج تلقائيًا في الواجهة.
- إذا أردت رفع الصور في Supabase Storage فاستخدم bucket اسمها `products` أو اترك الاسم كما هو إذا كانت موجودة بالفعل.

## التطوير التالي المنطقي

بعد إدخال المنتجات الأولى فعليًا في Supabase، أفضل خطوة لاحقة هي:
- إضافة صفحة/مسار مخصص لإعادة تعيين كلمة المرور
- أو تحسين لوحة الإدارة لتدير المنتجات والـ variants بدل الاقتصار على إدارة الطلبات فقط
