# الزّهراء / AL-ZHRAA NATURAL SOAP

مشروع storefront عربي RTL لبيع الصابون الطبيعي، مبني بـ HTML + CSS + JavaScript، ويعتمد على Supabase للمنتجات والطلبات ولوحة الإدارة.

## الحالة الحالية الفعلية

هذه النسخة هي النسخة العاملة الحالية للمشروع، وتعتمد على المسارات التالية:
- `index.html` + `script.js`: الصفحة الرئيسية وعرض المنتجات
- `cart.html` + `cart.js`: السلة وإرسال الطلب
- `admin-login.html` + `admin-orders.html` + `admin-orders.js`: تسجيل دخول الأدمن ولوحة الطلبات
- `style.css`: التنسيقات المشتركة

تم تنظيف هذا الـ README ليعكس المسار العامل فقط، بدون وصف legacy code قديم.

## المزايا المنفذة فعليًا

### المتجر
- المنتجات تُحمّل من جدول `products` في Supabase.
- صور المنتجات تُحمّل من Supabase Storage أو من رابط مباشر محفوظ في البيانات.
- يوجد فلترة للمنتجات.
- يوجد modal لعرض تفاصيل المنتج.
- يوجد شريط/أقسام ثقة وتحسينات واجهة وموبايل.

### السلة والطلب
- السلة في صفحة مستقلة: `cart.html`.
- إدارة السلة في `cart.js`.
- حفظ السلة في `localStorage` عبر المفتاح `soap-cart`.
- حفظ بيانات العميل في `localStorage` عبر المفتاح `soap-customer-info`.
- اختيار المحافظة من قائمة جاهزة لمحافظات مصر.
- حقل عنوان تفصيلي.
- حقل ملاحظات.
- خيار طلب مستعجل:
  - 5% من الإجمالي
  - بحد أدنى 10 جنيه
  - يظهر في الملخص
  - يُحفظ ضمن الطلب
  - يظهر في رسالة واتساب
- حفظ الطلب أولًا في جدول `orders` ثم فتح واتساب.
- استخدام `order_number` مع retry عند التكرار.
- ظهور شاشة نجاح بعد الطلب مع animation فقاعات.

### لوحة الإدارة
- تسجيل دخول الأدمن عبر Supabase Auth.
- عرض الطلبات من `orders_dashboard`.
- تحديث الحالة في `orders`.
- بحث.
- فلترة بالحالة.
- فلترة بالتاريخ.
- فلترة بالمحافظة.
- فرز.
- إحصائيات.
- إجمالي النتائج الحالية بعد الفلترة.
- تصدير CSV.
- نسخ رقم الطلب.
- زر واتساب سريع.
- عرض أوضح للعناوين والملاحظات ورسوم الاستعجال.

### SEO
- `robots.txt` موجود.
- `sitemap.xml` موجود.
- `canonical`, `og:*`, `twitter:*` موجودة في الصفحة الرئيسية.
- صفحات الإدارة `noindex`.
- صفحة السلة `noindex`، ولذلك لا تُدرج في `sitemap.xml`.

## الجداول المستخدمة في Supabase

### `products`
الواجهة العامة تقرأ المنتجات من هذا الجدول.

الحقول المستخدمة فعليًا في الكود:
- `id`
- `name`
- `price`
- `weight`
- `badge`
- `tag`
- `best_for`
- `category`
- `description`
- `long_description`
- `highlights`
- `benefits`
- `usage`
- `ingredients`
- `image`

### `orders`
يتم حفظ الطلبات فيه من `cart.js`، وتحديث الحالة منه في لوحة الإدارة.

### `orders_dashboard`
يُستخدم للعرض داخل لوحة الإدارة لأنه يجهز البيانات بصيغة مناسبة للقراءة والتقارير.

## تدفق الطلب الحالي

1. العميل يضيف المنتجات من الصفحة الرئيسية.
2. ينتقل إلى `cart.html`.
3. يراجع الكميات وبياناته.
4. عند الإرسال:
   - يتم التحقق من الاسم والهاتف والمحافظة والعنوان.
   - يتم حساب الإجمالي ورسوم الاستعجال إن وُجدت.
   - يتم إنشاء رقم طلب.
   - يتم حفظ الطلب في Supabase.
   - بعد نجاح الحفظ فقط يتم فتح رسالة واتساب.
   - ثم تظهر رسالة النجاح.

## إعدادات ثابتة مهمة

يوجد بالمشروع ثوابت مباشرة داخل ملفات JavaScript، مثل:
- `SUPABASE_URL`
- `SUPABASE_ANON_KEY`
- رقم واتساب
- قائمة المحافظات
- قواعد الطلب المستعجل

هذه الثوابت ما زالت موزعة على أكثر من ملف، وهي نقطة مناسبة لتوحيدها لاحقًا إذا أردت تحسين الصيانة.

## ملاحظات مهمة

- لا يوجد build step.
- لا يوجد npm أو framework.
- المشروع جاهز للنشر كـ static site.
- `script.js` الآن مخصص فقط لمسار الصفحة الرئيسية والمنتجات وشارة السلة، بدون منطق checkout قديم.
- منطق الطلب الفعلي كله موجود في `cart.js`.

## التطوير التالي المنطقي

أهم تطوير قادم بعد cleanup هو إدخال **منطق شحن فعلي** بأسعار أو قواعد حقيقية حسب المحافظة أو المنطقة، بدل الاكتفاء بعبارة أن الشحن يحدد لاحقًا.
